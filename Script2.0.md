# ARG 网页解密游戏 —— 剧本设计文档 2.0

> 最后更新：2026年6月2日

---

## 设计原则

- 所有核心内容在单一主题网站内完成
- 真实世界（小红书）仅作为发现入口
- 谜题为叙事服务，不为了难而难

---

## 一、游戏名称

### 《望秋》

双关含义：
- 沈望秋的名字
- "望向秋天"——走向终点前的一次回望
- "望秋而落"——在秋天到来之前凋零

---

## 二、玩家身份

**林念**，22岁，美术学院大四学生。

她的舅舅**陈序**是南山美术馆的馆长。她小时候经常去美术馆，记得有一位"沈叔叔"总会蹲下来跟她说话，教她在餐巾纸上画小动物。那些画她保留了很多年。

后来沈叔叔不来了。她问舅舅，舅舅说"他出差了"。

大学她考了美院，选择这个专业某种程度上也是因为小时候那些餐巾纸上的涂鸦。她现在经常去舅舅的美术馆闲逛、写生。

有一天，她在美术馆官网闲逛时，发现了一些异常——官网上似乎藏着一个不存在的展览的入口。

玩家以林念的第一人称视角，通过美术馆官网揭开这个隐藏展览的秘密。

> **为什么要强化林念的动机：** 她不只是"碰巧发现秘密"的旁观者。她小时候认识沈望秋，受过他的影响，如今即将毕业（面对自己的"告别"）。这让她的探索从好奇变成了"寻找一个记忆中的人"。

---

## 三、核心故事

### 3.1 表层故事

建筑师/装置艺术家**沈望秋**在50岁生日前夕自杀，骨灰撒入大海。他生前最大的愿望是——不留下任何痕迹。他花了几年时间销毁自己的作品：烧掉速写本、撕碎设计图、删除数字文件。

但他的挚友——**陈序**不忍心让这位才华横溢的设计师被世界彻底遗忘。葬礼后，他为每个精心准备好的礼盒拍了照，留作纪念，并在自己的美术馆中设置了一个隐藏的线上特展——希望至少有人能隐秘地记得这位伟大的设计师。

### 3.2 深层真相

先明确一件事：沈望秋为什么想自杀？

沈望秋在47岁那年被确诊为**早发性阿尔茨海默症**。

不是"可能"，不是"倾向"——确诊。他会逐渐失去记忆、失去认知、失去创作能力，最终失去自己。医生告诉他，5到8年内，他可能连身边的人都不认得了。

对一个靠头脑和双手创作的人来说，这是最残酷的判决。他亲眼见过自己的导师在病床上茫然地看着天花板，口齿不清，连自己的名字都写不出来。他不允许自己那样结束。

他决定：在还能控制自己命运的时候，体面地离开。

他只告诉了陈序一个人。

陈序的反应是拒绝接受。他拉着沈望秋跑遍了全国最好的医院，联系海外专家，查找实验性治疗方案，甚至组织朋友筹款。他拒绝相信"没有办法"。

这些，都是出自爱。

但对沈望秋来说，每一次"我再帮你找找别的医生"都在说同一句话："我接受不了你离开。"他没有力气再去安抚一个无法接受现实的好友了。他还要安排自己的告别——葬礼、礼物、以及怎样让最亲近的人慢慢接受这件事。

他销毁自己的作品，不是因为想消失——而是因为他正在经历自己的消失。他的大脑在背叛他。那些他曾经信手拈来的线条、空间构想、结构逻辑，正在一点点离他而去。他烧掉画稿，是因为看着自己画不出从前的水平，比什么都痛苦。

陈序从火里抢救那些速写本，不是因为不理解——而是因为他想抓住的从来不是那些纸，是那个还在的朋友。他救的不是画，是沈望秋。

但他们两个在做的，是两件相反的事：

- **沈望秋在练习告别。**
- **陈序在拒绝告别。**

两个人都爱对方。两个人都没错。但他们的错位，让沈望秋最后几年的负担更重了——他不仅要面对自己的疾病和死亡，还要承担好友的"不接受"。

他在礼物中埋下的真相，不是指控，而是解释：
*"我累了。不是因为病折磨我——是因为我无法让你放手。"*

## 四、道德困境

最终，林念拼凑出全部真相后发现：

陈序是真心爱他的朋友的。但他无法接受朋友的离去，用"帮忙"替代了"陪伴"，用"寻找治疗方案"替代了"坐在一起什么都不说"。他让沈望秋最后几年的告别变得无比沉重——因为沈望秋不仅要面对自己的死亡，还要不断地安抚一个不肯放手的朋友。

**林念要告诉舅舅吗？** 告诉他：你的爱让他在最后的日子里更累了？

**两个结局路径（玩家选择）：**

- **选择告诉** → 陈序沉默很久，说"我知道。我只是……不想让他走。" → 画面渐暗。**没有和解，只有面对。**
- **选择不告诉** → 林念合上电脑。她决定让舅舅继续保持他对朋友的理解。有些真相不会让任何人更好过。 → 画面渐暗。**没有隐瞒，只有不忍。**

没有"正确"答案。故事停在选择后的一小段沉默里。

## 五、整体流程

```
小红书 → 发现可疑线索（照片角落里的纸片）
    │
    ▼
南山美术馆官网（看似正常的美术馆网站）
    │ 小红书上的线索指向隐藏路径
    ▼
"望秋"特展入口 / 开门谜题
    │ 破解入口密码
    ▼
策展人序言（陈序的文字 → 自我认知："我是为了记住他"）
    │
    ▼
四件礼物的照片 + 陈序的注解
（注解中充满了"这是我替他保住的"、"幸好我及时捡了回来"）

    ├─ 礼物#1 → 破解 → 获得碎片#1
    ├─ 礼物#2 → 破解 → 获得碎片#2
    ├─ 礼物#3 → 破解 → 获得碎片#3
    └─ 礼物#4 → 破解 → 获得碎片#4
    │
    ▼
四个碎片合并
    │
    ▼
真相页 → 拼出"被抢救的遗稿"背后的故事
         → 揭示陈序无意识中反复触碰的边界
         → 林念面临选择（告诉 / 不告诉）
```

## 六、双层网站设计

### 6.1 第一层：南山美术馆官网（公开页面）

一个看起来功能完整的美术馆网站。

| 页面 | 内容 |
|------|------|
| 首页 | 美术馆介绍、当前展览轮播、参观信息 |
| 展览 | 当前展览列表 + 过往展览归档 |
| 关于 | 美术馆简介、馆长介绍（陈序） |
| 参观 | 开放时间、地址、导览图 |

**隐藏入口位置：**

"展览" → "过往展览"归档中，有一条记录：

> **"未命名特展（筹备中）"**

链接指向 `/exhibitions/pending`

该页面显示：**"此展览尚未开放"**

页面源代码中有一行注释：

```html
<!-- 地址是对的，入口是错的。再想想。 -->
```

真正的入口路径线索在小红书上。

## 七、入口发现（小红书线索 — 具体设计）

### 场景

林念发了一组在南山美术馆闲逛时拍的照片，共 4 张：

1. 美术馆中庭阳光洒下来的样子
2. 自己在画架前写生的背影
3. 舅舅办公室门开着，桌上堆满文件（这张是偷拍视角，带点调皮）
4. 一张随手拍——她靠在走廊的墙边，对面是一台开着的电脑，屏幕上隐约是美术馆网站后台的某个页面

### 关键的细节

在 **第3张照片** 中：舅舅办公桌的文件堆里，露出一角打印纸，上面写着：

```
展览路径：/exhibitions/wangqiu
密码提示：他的生日 + 我的生日
```

PS: 用八位数字。

---

在 **第4张照片** 中：林念手机屏幕（她在拍自己的影子）反射出电脑屏幕的一小块，上面是 Chrome 开发者工具，高亮了一行 HTML 注释：

```html
<!-- 地址是对的，入口是错的。再想想。 -->
```

### 发布方式

- 小红书账号 @林念念 发帖："在舅舅的美术馆泡了一下午 ☕️"
- 图片本身看起来完全正常
- 只有极其细心的玩家才会放大观察文件堆里的那行字和屏幕反射
- 评论区如果有猜测，林念不会回复（这是设定，非玩家）

## 八、开门谜题（入口验证）

### 页面

`/exhibitions/wangqiu`

一个极简的页面：深色背景，中央有一行字：

> **"未命名特展"**

下方是一个密码输入框，提示文字：

> 请输入八位数字密码

### 密码答案

**19730526** —— 沈望秋的生日（1973年5月26日）

### 线索来源

小红书照片中纸片上写的是：
> 他的生日 + 我的生日

但玩家需要自己查——"他的生日"是哪一天？

在美术馆官网的 **"关于"** 页面中，有一个 **"过往策展团队"** 列表，其中沈望秋的名字带有生卒年份：
> 沈望秋（1973-2026）

而"我的生日"（陈序的生日）在 **馆长介绍** 中可见：5月26日。

组合方式：从小红书照片中纸片上"八位数字"的提示，玩家需推断格式为 **YYYYMMDD**。

### 错误反馈

- 输入错误密码 → 页面提示："密码错误。再想想，入口就在你眼前。"
- 连续错误 5 次 → 提示变为一段引用（沈望秋的话）：
  > "有些门要试很多次才能打开。但有些门，打开之前最好想清楚要不要进去。"

## 九、四件礼物（详细设计）

### 设定前提

葬礼上，沈望秋为四位特定宾客各准备了一件回赠礼物。这四件礼物是他确诊之后，在还能清晰思考和创作时，亲手准备的。

陈序在礼物被带走前逐一拍照，并配上自己的注解。他以为这些照片记录了"朋友最后的温暖"，但他不知道，这些礼物中还藏着另一个故事——关于他始终不肯面对的那个真相。

---

### 礼物#1 —— 一幅装裱好的素描

| 项目 | 内容 |
|------|------|
| 送给谁 | 美院时期的恩师 |
| 照片内容 | 一幅人物素描，画的是沈望秋年轻时在画室的自画像，装裱精细，右下角有签名和日期 |
| 陈序的注解 | "这是他刚入学时的作品。构图很扎实，那时候他线条真利落。他说要送给老师，我帮他裱的。" |
| 隐藏线索 | 素描的签名日期旁，有一行极小的铅笔字（放大可见）：**"2023.6.3 确诊。还能画最后一张。"** |
| 碎片文字 | "2023年6月3日，我拿到确诊报告。第一个告诉他，他说：'一定是误诊，我们再查。'" |

**交互方式：** 玩家可以点击图片放大，在签名区域通过放大镜工具找到那行极小的铅笔字。

> **深化设计：** 铅笔字的前半句"2023.6.3 确诊"字迹相对工整，后半句"还能画最后一张"开始轻微颤抖——玩家第一次发现时可能忽略这个区别，但在回顾时会意识到：**他在写下这句话的时候，手已经开始抖了。**

**陈序注解的语气：** 平静、就事论事。他还在把素描当作"一幅画"来评价。

---

### 礼物#2 —— 一张病历单（复印件）

| 项目 | 内容 |
|------|------|
| 送给谁 | 前女友（护士） |
| 照片内容 | 一张被折过的病历单的局部，字迹隐约可见，看起来像是一份医疗报告 |
| 陈序的注解 | "这张纸他夹在一本书里还给了我。我问过他是什么，他说以前的体检报告。我没多想。" |
| 隐藏线索 | 病历单上的部分文字被涂黑了。但对着光看，可以辨认出被涂黑的词：**"早发性阿尔茨海默症"**、**"认知功能进行性下降"** |
| 碎片文字 | "他带我去最好的医院复查，挂了五个专家号。每个都说了同样的话。他把报告收起来，说'他们都不够权威'。" |

**交互方式：** 滑块调节光线角度，让纸张背光透出被涂黑的文字。也可以加入"紫外线光"模式（页面角落有一个小图标，点击切换光源）。

> **深化设计：** 在正常光线下，病历单看起来只是一份普通的体检报告。切换到背光模式后，被涂黑的诊断结论浮现——但更令人不安的是：纸张背面有手写的几行字（沈望秋的字），在背光下隐约可见，内容是他在不同日期记下的症状：
> - "9.15 忘了回家的路，在小区里走了四十分钟"
> - "11.2 想不起她的名字"
> - "1.7 今天画不出一个正方形"
>
> 这些字逐行变得更抖。最后一行的"正方形"三个字几乎无法辨认。

**陈序注解的语气：** 开始有了一丝不确定。"我没多想"三个字透露出他在回避某些直觉。

---

### 礼物#3 —— 一本日历

| 项目 | 内容 |
|------|------|
| 送给谁 | 前助理 |
| 照片内容 | 一本2024年的桌面日历，摊开在12月那一页，上面有很多手写的标记 |
| 陈序的注解 | "他有记日程的习惯。这本日历我一直留着。翻到后面……他好像下半年没那么忙了。" |
| 隐藏线索 | 日历上的标记逐渐变得混乱——前半年的字迹清晰工整，后半年的字迹开始抖动、有涂改、有时连续几天只画了一条线而不是写字。每一页的角落都有一个很小的数字，是日期的倒计时 |
| 碎片文字 | "他在日历上画了最后一笔的那天，是他决定不再去看新医生的日子。" |

**交互方式：** 逐月翻动日历（点击翻页），观察字迹的变化——这是一个渐进式发现的谜题，也是一段无声的纪录。

| 月份 | 字迹状态 | 内容特征 |
|------|----------|----------|
| 1-3月 | 工整 | 排满日程：复诊、见专家、创作计划 |
| 4-6月 | 偶有颤抖 | 开始有"忘记带钥匙"、"走错路"等备注 |
| 7-9月 | 明显抖动 | 日程变少，多次出现空白日，"取消"标记增多 |
| 10-12月 | 近乎难以辨认 | 大片空白，偶尔只有一条颤抖的横线 |

倒计时数字从 365（2024.1.1）开始，到 1（2024.12.31），然后有一页被撕掉了。

> **深化设计：** 撕掉的那一页是 2025年1月1日。在页面的缝隙中，隐约能看到残留的字迹——只有一个词，是沈望秋写给自己的：**"够了。"**

**陈序注解的语气：** 他试图轻描淡写（"好像下半年没那么忙了"），但这句话本身就是一种刻意的淡化——他开始意识到有些不对，但不愿意深究。

---

### 礼物#4 —— 一封未寄出的信

| 项目 | 内容 |
|------|------|
| 送给谁 | **陈序** |
| 照片内容 | 一封信，装在空白信封里，没有署名收件人。信纸从信封口露出一角 |
| 陈序的注解 | "他走了以后，我在他桌上发现了这封信。没有收件人……不知道是写给谁的。我一直没打开。" |
| 隐藏线索 | 信纸露出的那一角有几行字。通过放大可以辨认出开头几个字——**"序哥：读到这封信的时候……"**。原来收件人是陈序自己，但他从来没有打开看过。 |
| 碎片文字 | "序哥：读到这封信的时候，我应该已经不在了。你一定在怪自己——怪自己没能拉住我。但我想让你知道：你拉了我三年……" |

**交互方式：** 通过拖拽信封口，逐渐将信纸拉出（模拟动画），逐行显示信的内容。

> **深化设计：** 信纸拖出的过程分为两段：
> - **第一段（前 2/3）：** 文字看起来是一封普通的告别信——理性、克制、告别。
> - **停顿点：** 拖到这里，信纸卡了一下。玩家需要再次拖拽才能拉出剩余部分。
> - **第二段（后 1/3）：** 字迹突然变得很抖，内容是沈望秋最真实的情感——不是解释，而是疲惫。
>
> 这个停顿是设计上的"呼吸点"，让玩家在阅读后 1/3 之前有一个心理准备的间隙。

**陈序注解的语气：** 极其简短，有两处欲言又止：
- "没有收件人……"（省略号代表他的犹豫——他其实隐约知道是写给谁的）
- "我一直没打开。"（这是一句对自己的辩解，他不敢打开）

## 十、真相的完整拼图

当四件礼物全部解锁后，最终页展示：

### 碎片组合成的完整叙述

**碎片#1（素描）：**
2023年6月3日确诊。沈望秋第一个告诉了陈序。陈序说："一定是误诊，我们再查。"

**碎片#2（病历单）：**
陈序带他跑了十三家医院、挂了七个专家号。每个都说了一样的话。陈序把报告收起来，说"他们都不够权威"。

**碎片#3（日历）：**
日历上的字迹从清晰到颤抖，从颤抖到空白。最后一笔停留的那天，沈望秋决定不再去看新医生了——不是因为他放弃了，是因为他太累了。

**碎片#4（信）：**
那封信的收件人是陈序。沈望秋想告诉他——

*"你让我多活了三年。但我真的累了。"*

### 页面最终的呈现

页面上缓缓出现那封信的完整内容（模拟沈望秋的手写字迹，字迹从工整到轻微颤抖）：

> 序哥：
>
> 读到这封信的时候，我应该已经不在了。
>
> 你一定在怪自己——怪自己没能拉住我。
>
> 但我想让你知道：你拉了我三年。
>
> 你带我看了十三个医生，找了七个专家，联系了三个国家的实验室。你拒绝接受这件事，坚持了三年。
>
> 这三年里我每次想放弃，看到你的脸，我就再多撑了一天。
>
> 不是你拖累了我。**是你让我多活了三年。**
>
> 但我真的累了。让我休息吧。
>
> 下辈子不做你朋友了——太累了。做你养的猫吧，每天除了睡就是吃。

（最后一行字明显更抖，像是写到这里已经费尽了力气。但即便如此，他还在开玩笑。）

页面下方，一行字淡入：

> **林念合上了电脑。她知道了所有的事。**
>
> **她要把这封信给舅舅看吗？**

### 玩家选择

页面上出现两个选项，没有对错标签：

| 选项 | 文字 | 后续 |
|------|------|------|
| **告诉** | "舅舅，我在你电脑上看到了一个展览……有些事，你应该知道。" | → 陈序沉默很久。他说："我知道。我一直都知道。我只是……不想让他走。" → 画面渐暗。**没有和解，只有面对。** |
| **不告诉** | 林念删除了浏览记录，合上电脑。有些真相，不需要被所有人知道。 | → 画面渐暗。最后一行字："她保留了那个秘密。不是因为懦弱，是因为爱有时需要沉默。" → **没有隐瞒，只有不忍。** |

## 十一、陈序的角色定位总结

| 层面 | 说明 |
|------|------|
| 他对设计师的感情 | 真实的、深厚的友情。他确实爱他的朋友 |
| 他办展的动机 | 纯粹——他不忍心朋友被完全遗忘，想让人记得他 |
| 他在悲剧中的角色 | 他拒绝接受朋友患病的事实，用"找更好的医生"替代了"陪他走最后一程"。他的不放手让沈望秋最后三年的负担更重——不仅要面对自己的消失，还要安抚一个无法告别的好友 |
| 他对自己的认知 | 他觉得自己在救朋友——只要还没放弃，就有希望 |
| 真相的核心 | 真相不是"他被背叛了"，而是"他被爱得太用力了，以至于无法轻松地离开" |
| 他是否应该知道真相 | 信中说得很清楚："不是你拖累了我，是你让我多活了三年。" 但林念读到的是完整版——她知道舅舅的"不放手"客观上增加了负担。她要不要让舅舅知道？ |

陈序不是恶人。他甚至做得比大多数朋友都好——三年里不放弃、带他看遍名医、拒绝接受死亡。但有时候，"不放弃"对需要"被允许离开"的人来说，是一种沉重的爱。

**这是整个故事的主题。**

## 十二、技术实现要点

| 项目 | 说明 |
|------|------|
| 页面类型 | 纯前端静态页面（推荐 React / Vue SPA，或原生 HTML + CSS + JS） |
| 路由 | 前端路由模拟多页面，实际上所有内容在同一个站点内 |
| 交互组件 | 放大镜（Canvas 实现区域放大）、滑块（光线角度控制）、日历翻页（点击切换月份）、拖拽拉信纸（鼠标/触屏拖拽） |
| 移动端适配 | 所有交互需支持触屏（拖拽、缩放、点击翻页） |
| 隐藏注释 | HTML 注释中的线索需确保不被构建工具清除（需在 build 配置中保留） |
| 进度保存 | 建议使用 localStorage 保存玩家进度（已解锁哪些礼物） |
| 字体 | 信的内容需使用手写体中文网页字体（如"站酷仓耳渔阳体"等可商用字体） |
